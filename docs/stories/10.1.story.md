## Story

**As a** jardineiro que acessa o sistema pelo computador,
**I want** fazer login no dashboard web com minha conta Google ou telefone,
**so that** eu possa gerenciar meu negocio pela tela grande quando estiver em casa/escritorio.

## Acceptance Criteria

```gherkin
AC-1: Login Web com Google
GIVEN o usuario acessa o dashboard web
WHEN ele clica em "Entrar com Google"
THEN o fluxo OAuth e iniciado via Supabase Auth
AND apos autorizacao, ele e redirecionado para o dashboard "/"
AND a sessao e mantida via cookies httpOnly

AC-2: Login Web com Telefone (OTP)
GIVEN o usuario acessa a pagina de login
WHEN ele informa o telefone e recebe o codigo OTP
THEN apos validar o codigo, ele e redirecionado para o dashboard "/"

AC-3: Layout Base com Sidebar
GIVEN o usuario autenticado
WHEN ele navega pelo dashboard
THEN ele ve uma sidebar fixa a esquerda com links: Dashboard, Clientes, Agenda, Orcamentos, Financeiro, Equipe, Configuracoes
AND um header com nome do usuario, avatar e botao de logout
AND a sidebar e responsiva (colapsa em mobile/tablet)

AC-4: Protecao de Rotas
GIVEN um usuario nao autenticado
WHEN ele tenta acessar qualquer rota do dashboard
THEN ele e redirecionado para "/login"

AC-5: Logout
GIVEN o usuario autenticado
WHEN ele clica em "Sair"
THEN a sessao e destruida e ele e redirecionado para "/login"
```

[Source: PRD Section 1.4 (Wave 2), architecture.md#2.2, #2.4]

- [ ] **Task 7** (AC: 5): Logout server action

  - Server action que chama `supabase.auth.signOut()`
