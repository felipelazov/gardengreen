## Story

**As a** jardineiro que acabou um servico,
**I want** marcar como concluido com 1 toque,
**so that** mantenha meu registro atualizado e possa cobrar.

## Acceptance Criteria

```gherkin
AC-1: Concluir servico (swipe ou botao)
GIVEN um servico agendado para hoje
WHEN o usuario faz swipe para direita no card do servico OU toca no botao "Concluir"
THEN o status muda para "Concluido" com animacao de check verde (< 300ms)
AND a hora de conclusao e registrada automaticamente (timestamp UTC)
AND o sistema exibe prompt "Cobrar agora?" com link para fluxo de cobranca

AC-2: Desfazer conclusao
GIVEN um servico recem marcado como concluido
WHEN o usuario ainda esta na janela de 5 segundos
THEN a snackbar exibe botao "Desfazer"
AND ao tocar, o status volta para "Agendado"
AND apos 5 segundos, a reversao so e possivel via edicao do servico
```

[Source: PRD Section 8, EPIC-02, Story 2.4]
