## Story

**As a** jardineiro logado no dashboard web,
**I want** ver um resumo do meu dia e metricas importantes na tela inicial,
**so that** eu saiba rapidamente o que preciso fazer hoje e como esta meu negocio.

## Acceptance Criteria

```gherkin
AC-1: Agenda do Dia
GIVEN o usuario acessa o dashboard "/"
WHEN a pagina carrega
THEN ele ve os servicos agendados para hoje em cards ordenados por horario
AND cada card mostra: cliente, endereco, horario, tipo de servico, status
AND servicos concluidos ficam com visual diferenciado (opacity reduzida, check)

AC-2: Metricas Resumidas
GIVEN o usuario acessa o dashboard
WHEN a pagina carrega
THEN ele ve 4 cards de metricas no topo:
  - Receita do mes (R$)
  - Servicos realizados no mes
  - Clientes ativos
  - Orcamentos pendentes
AND cada card mostra valor atual e variacao percentual vs mes anterior

AC-3: Alertas e Pendencias
GIVEN existem itens que requerem atencao
WHEN o dashboard carrega
THEN uma secao "Atencao" mostra:
  - Cobrancas vencidas (clientes inadimplentes)
  - Orcamentos aguardando resposta ha mais de 7 dias
  - Servicos sem confirmacao para amanha
AND cada alerta e clicavel e leva para a secao correspondente

AC-4: Estado Vazio
GIVEN o usuario nao tem servicos agendados hoje
WHEN o dashboard carrega
THEN mostra mensagem motivacional: "Sem servicos hoje — aproveite para enviar orcamentos!"
AND sugere acoes: "Criar orcamento", "Agendar servico"
```

### Queries Supabase

```typescript
// Receita do mes
const { data } = await supabase
  .from('services')
  .select('price')
  .eq('user_id', userId)
  .eq('status', 'completed')
  .gte('scheduled_date', startOfMonth)
  .lte('scheduled_date', endOfMonth);

// Clientes ativos
const { count } = await supabase
  .from('clients')
  .select('*', { count: 'exact', head: true })
  .eq('user_id', userId)
  .eq('status', 'active');
```
