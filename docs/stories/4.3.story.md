## Story

**As a** jardineiro,
**I want** converter um orcamento aprovado em servico agendado,
**so that** nao precise redigitar as informacoes que ja estao no orcamento.

## Acceptance Criteria

```gherkin
AC-1: Botao disponivel apenas para orcamentos aprovados
GIVEN um orcamento com status "Aprovado"
WHEN o jardineiro visualiza o detalhe do orcamento
THEN o botao "Agendar Servico" esta visivel e ativo

AC-2: Pre-preencher formulario de servico
GIVEN o jardineiro toca em "Agendar Servico"
WHEN a tela de criacao de servico e aberta
THEN o cliente ja esta selecionado
AND o valor total do orcamento e pre-preenchido como valor do servico
AND o tipo de servico e pre-definido como "orcamento"
AND a descricao inclui o numero do orcamento

AC-3: Link bidirecional
GIVEN o servico e criado a partir do orcamento
WHEN o servico e salvo
THEN o orcamento referencia o ID do servico criado
AND o servico referencia o ID do orcamento de origem

AC-4: Status do orcamento atualizado
GIVEN o servico foi criado com sucesso
WHEN o usuario volta para o detalhe do orcamento
THEN o orcamento exibe "Servico agendado" com link para o servico
```

## Dev Notes

### Fluxo de Conversao

1. Orcamento aprovado → botao "Agendar Servico"
2. `useConvertToService(quoteId)` executa:
   - Busca dados do orcamento (items, client_id, total)
   - Cria servico via `useCreateService`
   - Atualiza orcamento: `service_id = novo servico ID`
3. Navega para detalhe do servico criado

### Mapeamento de Campos

| Campo Orcamento | Campo Servico |
|-----------------|---------------|
| `client_id` | `client_id` |
| `total` (centavos) | `value` (centavos) |
| `number` | `notes` (prefixo "Orcamento ORC-XXX") |
| — | `type` = "servico" |
| — | `date` = hoje (usuario edita) |
