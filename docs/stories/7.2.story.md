## Story

**As a** jardineiro autônomo,
**I want** receber uma notificação à noite com meus clientes de amanhã,
**so that** eu me organize antes de dormir sem precisar verificar o app.

## Acceptance Criteria

```gherkin
AC-1: Notificacao enviada no horario correto
GIVEN usuarios com evening_enabled=true em notification_preferences
WHEN o cron executa as 19:30 BRT
THEN cada usuario recebe um push com os clientes agendados para amanha

AC-2: Conteudo da mensagem
GIVEN um usuario com servicos agendados para amanha
WHEN o push e enviado
THEN o titulo e "Agenda de Amanha"
AND o corpo e "Amanha voce tem [X] clientes. Primeiro as [hora]: [nome do primeiro]"

AC-3: Sem servicos amanha — sem push
GIVEN um usuario sem servicos agendados para amanha
WHEN o cron executa
THEN nenhuma notificacao e enviada para esse usuario

AC-4: Tap na notificacao abre agenda de amanha
GIVEN o usuario recebeu o push de vespera
WHEN ele toca na notificacao
THEN o app abre direto na tela de agenda filtrada para amanha

AC-5: Token invalido — ignorado silenciosamente
GIVEN um usuario sem expo_push_token valido
WHEN o cron executa
THEN o usuario e pulado sem erro critico
```

[Source: PRD EPIC-07, Story 7.2]
