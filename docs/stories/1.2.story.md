## Story

**As a** jardineiro que nao usa email,
**I want** entrar no app usando apenas meu numero de celular,
**so that** tenha acesso rapido sem complicacao.

## Acceptance Criteria

```gherkin
AC-1: Envio de OTP
GIVEN o app aberto na tela de login
WHEN o usuario toca em "Entrar com telefone"
AND digita seu numero com DDD (ex: 11999887766)
THEN o sistema envia SMS com codigo de 6 digitos via Supabase Auth
AND exibe tela para digitar o codigo

AC-2: Verificacao de OTP (Happy Path)
GIVEN o codigo SMS recebido
WHEN o usuario digita o codigo correto dentro de 5 minutos
THEN o login e realizado com sucesso
AND usuario novo vai para onboarding, usuario existente vai para tela principal

AC-3: Bloqueio por tentativas
GIVEN o codigo digitado incorreto 3 vezes
WHEN o usuario tenta novamente
THEN o sistema bloqueia tentativas por 5 minutos
AND exibe mensagem: "Muitas tentativas. Aguarde 5 minutos."

AC-4: Reenvio de codigo
GIVEN o usuario na tela de verificacao
WHEN ele nao recebeu o SMS em 60 segundos
THEN o botao "Reenviar codigo" e habilitado
AND ao tocar, envia novo SMS e reinicia o timer

AC-5: Formato brasileiro
GIVEN o usuario digitando o telefone
WHEN ele digita os numeros
THEN a mascara brasileira e aplicada automaticamente: (11) 99988-7766
AND o numero e salvo internamente no formato +5511999887766
```

## Dev Notes

### Dados do PRD

- **FR-002**: Autenticacao via telefone (OTP/SMS)
- Story Points: 5
- Custo SMS: considerar que Supabase usa Twilio (~US$0.0075/SMS)
- Limite recomendado: 5 SMS/dia por numero

### Arquitetura Relevante

**Supabase Phone Auth:**
- `signInWithOtp({ phone })` envia SMS via Twilio (configurado no Supabase Dashboard)
- `verifyOtp({ phone, token, type: 'sms' })` valida codigo
- Rate limit: 30 requests/hora por IP (Supabase default)
- OTP expira em 5 minutos (default Supabase)
[Source: architecture.md#11.2 Fluxo de Autenticacao]

**Trigger automatico:**
- O trigger `handle_new_user()` roda automaticamente apos primeiro login
- Cria registro em `users` com `phone` do auth e nome default "Jardineiro"
- Cria `notification_preferences` com defaults
- O app NAO precisa fazer INSERT manual
[Source: architecture.md#9.1 DDL — handle_new_user]

**Validacao de telefone (Zod):**
```typescript
// packages/shared/src/schemas/phoneSchema.ts
import { z } from 'zod';

export const phoneSchema = z.string()
  .regex(/^\+55\d{10,11}$/, 'Telefone invalido. Use formato com DDD.');

export const formatPhoneForDisplay = (phone: string): string => {
  // +5511999887766 → (11) 99988-7766
  const digits = phone.replace(/\D/g, '').slice(2); // remove +55
  return `(${digits.slice(0, 2)}) ${digits.slice(2, 7)}-${digits.slice(7)}`;
};
```
[Source: architecture.md#6.3 Shared Package — schemas]
