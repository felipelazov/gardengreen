## Story

**As a** jardineiro usando o dashboard web,
**I want** criar, visualizar e gerenciar orcamentos em uma interface organizada,
**so that** eu possa enviar propostas profissionais e acompanhar respostas dos clientes.

## Acceptance Criteria

```gherkin
AC-1: Lista de Orcamentos
GIVEN o usuario acessa "/orcamentos"
WHEN a pagina carrega
THEN ele ve uma tabela com colunas: Cliente, Descricao, Valor, Status, Data de Envio, Validade
AND filtros por status: Todos, Rascunho, Enviado, Aprovado, Rejeitado, Expirado
AND ordenacao padrao: mais recentes primeiro

AC-2: Criar Orcamento
GIVEN o usuario na pagina de orcamentos
WHEN ele clica em "Novo Orcamento"
THEN abre um formulario com:
  - Cliente (combobox searchable)
  - Itens do orcamento (lista dinamica): descricao, quantidade, valor unitario, subtotal
  - Observacoes gerais
  - Validade (dias, default 30)
  - Total calculado automaticamente
AND validacao Zod (`quoteSchema`)

AC-3: Enviar via WhatsApp
GIVEN um orcamento criado
WHEN o usuario clica em "Enviar WhatsApp"
THEN gera mensagem formatada com itens e total
AND abre WhatsApp Web com a mensagem pre-preenchida para o telefone do cliente
AND o status muda para "Enviado"

AC-4: Acoes em Batch
GIVEN orcamentos selecionados via checkbox
WHEN o usuario seleciona uma acao em batch
THEN pode: Marcar como Aprovado, Marcar como Rejeitado, Excluir rascunhos
AND confirmacao antes de executar

AC-5: Duplicar Orcamento
GIVEN um orcamento existente
WHEN o usuario clica em "Duplicar"
THEN cria uma copia como rascunho com data atualizada
AND permite editar antes de salvar
```

[Source: PRD FR-006/007, architecture.md#4.1 Quote interface]
