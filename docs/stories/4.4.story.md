## Story

**As a** jardineiro,
**I want** gerenciar o status dos meus orcamentos com cores distintas,
**so that** consiga visualizar rapidamente quais orcamentos precisam de atencao.

## Acceptance Criteria

```gherkin
AC-1: Status com cores distintas
GIVEN um orcamento em qualquer status
WHEN o badge de status e exibido
THEN as cores sao:
  - Rascunho: cinza (#6B7280 / #F3F4F6)
  - Enviado: azul (#2563EB / #DBEAFE)
  - Aprovado: verde (#15803D / #DCFCE7)
  - Recusado: vermelho (#DC2626 / #FEE2E2)
  - Expirado: laranja (#D97706 / #FEF3C7)

AC-2: Filtros por status na listagem
GIVEN a tela de listagem de orcamentos
WHEN o usuario seleciona um filtro (chip)
THEN apenas os orcamentos daquele status sao exibidos
AND o chip ativo e destacado em verde

AC-3: Marcar como Aprovado
GIVEN um orcamento com status "Enviado"
WHEN o jardineiro toca em "Marcar Aprovado"
THEN o status muda para "Aprovado"
AND a data de aprovacao e registrada

AC-4: Marcar como Recusado
GIVEN um orcamento com status "Enviado"
WHEN o jardineiro toca em "Marcar Recusado"
THEN uma confirmacao e solicitada
AND apos confirmar, o status muda para "Recusado"

AC-5: Expiracao automatica
GIVEN um orcamento com status "Enviado" cuja data de validade passou
WHEN o usuario abre o app
THEN o status e atualizado automaticamente para "Expirado"
```

[Source: PRD Section EPIC-04, Story 4.4]
