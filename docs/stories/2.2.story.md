## Story

**As a** jardineiro com clientes fixos,
**I want** configurar servicos recorrentes (ex: toda terca no Sr. Joao),
**so that** nao precise agendar manualmente toda semana.

## Acceptance Criteria

```gherkin
AC-1: Criar recorrencia
GIVEN o usuario criando um novo servico
WHEN ele ativa o toggle "Repetir" e seleciona frequencia (semanal/quinzenal/mensal)
AND define o dia da semana
THEN o sistema cria servicos automaticos para as proximas 8 semanas
AND cada servico pode ser editado/cancelado individualmente sem afetar os demais

AC-2: Editar servico mestre
GIVEN servicos recorrentes ja criados
WHEN o usuario edita o servico "mestre" (template da recorrencia)
THEN o sistema pergunta: "Alterar apenas este ou todos os futuros?"
AND aplica a escolha do usuario

AC-3: Renovacao automatica
GIVEN o periodo de 8 semanas de instancias chegando ao fim
WHEN faltam 2 semanas para acabar
THEN o sistema gera mais 8 semanas automaticamente
AND notifica o usuario apenas se houver conflitos de horario
```

[Source: PRD Section 8, EPIC-02, Story 2.2]

## Dev Notes

- **FR-006**: Servicos recorrentes com frequencia configuravel
- **Estrategia de instancias**: gerar registros concretos (NAO calcular on-the-fly) — facilita edicao individual e sync offline
- **Frequencias suportadas**: `'semanal' | 'quinzenal' | 'mensal'`
- `Recurrence` e o "template mestre"; cada `Service` gerado e uma instancia independente
- [Source: PRD Technical notes Story 2.2, architecture.md#10.4 Offline-First]
