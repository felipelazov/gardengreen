## Story

**As a** jardineiro com cobranças pendentes,
**I want** receber lembretes automaticos sobre pagamentos nao recebidos,
**so that** possa fazer o acompanhamento sem precisar verificar manualmente.

## Acceptance Criteria

```gherkin
AC-1: Lembrete automatico apos N dias
GIVEN existe pagamento com status=pending
AND o pagamento esta pendente ha N dias (configuravel: 1, 3, 5 ou 7 dias)
WHEN o cron job diario executa
THEN o jardineiro recebe push notification: "Cobranca pendente: [Cliente] - R$ [Valor]"
AND a notificacao e enviada apenas uma vez por ciclo de N dias

AC-2: Intervalo configuravel
GIVEN o jardineiro acessa as configuracoes
WHEN ele define o intervalo de lembrete (1, 3, 5 ou 7 dias)
THEN o sistema respeita o novo intervalo para lembretes futuros

AC-3: Nao duplicar lembretes
GIVEN um lembrete ja foi enviado para um pagamento nas ultimas 24h
WHEN o cron job executa novamente
THEN nenhum novo lembrete e enviado para esse pagamento

AC-4: Ignorar pagamentos resolvidos
GIVEN um pagamento e marcado como paid ou overdue
WHEN o cron job executa
THEN nenhum lembrete e enviado para esse pagamento
```

### Logica de Cron

```typescript
// Pseudocodigo da Edge Function
const pendingPayments = await supabase
  .from('payments')
  .select('*, users(push_token, reminder_interval_days)')
  .eq('status', 'pending')
  .is('paid_at', null);

for (const payment of pendingPayments) {
  const daysPending = diffInDays(today, payment.created_at);
  const interval = payment.users.reminder_interval_days ?? 3;
  const lastReminder = payment.last_reminder_at;

  if (daysPending % interval === 0 && !sentInLast24h(lastReminder)) {
    await sendPushNotification(token, 'Cobranca pendente', `${clientName} - R$ ${amount}`);
    await updateLastReminderAt(payment.id);
  }
}
```

### Configuracao do Cron

Usar `pg_cron` no Supabase (disponivel em planos pagos) ou webhook externo (GitHub Actions / Render cron):
