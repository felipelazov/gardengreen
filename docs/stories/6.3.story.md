## Story

**As a** jardineiro autonomo,
**I want** receber automaticamente um relatorio financeiro no inicio de cada mes,
**so that** possa entender meu desempenho sem precisar calcular nada manualmente.

## Acceptance Criteria

```gherkin
AC-1: Geracao automatica no dia 1
GIVEN e o dia 1 de um novo mes as 03:00 BRT
WHEN a Edge Function `generate-monthly-report` e executada via cron
THEN para cada usuario com onboarding_completed=true
AND gera relatorio com: receita total, despesas, lucro liquido, servicos concluidos, clientes atendidos, pendentes
AND insere registro na tabela `monthly_reports`

AC-2: Notificacao push
GIVEN o relatorio foi gerado com sucesso
WHEN o usuario tem push notifications habilitado
THEN recebe: "Seu relatorio de [Mes] esta pronto! Voce ganhou R$ X.XXX"

AC-3: Tela de relatorio
GIVEN o jardineiro recebe a notificacao ou navega para o relatorio
WHEN abre a tela `report/[id]`
THEN ve: cabecalho com mes/ano, grandes numeros de receita/despesas/lucro
AND ve: servicos concluidos, clientes atendidos, valor pendente destacado
AND ve: grafico de barras comparando os ultimos 3 meses (usando Views, sem biblioteca externa)

AC-4: Compartilhar relatorio
GIVEN o jardineiro visualiza o relatorio
WHEN toca em "Compartilhar"
THEN o app formata o relatorio como texto estruturado
AND abre o seletor nativo de compartilhamento (Share API)
AND o texto inclui todos os numeros principais formatados em PT-BR

AC-5: Grafico dos ultimos 3 meses
GIVEN o jardineiro tem historico de pelo menos 2 meses
WHEN visualiza o relatorio
THEN ve barras verticais proporcionais para cada mes com receita e despesa
AND cada barra tem o valor exibido acima
AND meses sem dados exibem barra vazia com label "Sem dados"
```

## Dev Notes

### Edge Function

- Linguagem: TypeScript (Deno runtime no Supabase)
- Autenticacao: service_role key (acesso total ao banco)
- Cron: `0 6 1 * *` (06:00 UTC = 03:00 BRT)
- Tabela de destino: `monthly_reports` (criar migration se nao existir)

### Estrutura da tabela monthly_reports

```sql
CREATE TABLE monthly_reports (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL REFERENCES users(id),
  month integer NOT NULL,
  year integer NOT NULL,
  total_revenue integer NOT NULL DEFAULT 0,
  total_expenses integer NOT NULL DEFAULT 0,
  net_profit integer NOT NULL DEFAULT 0,
  services_completed integer NOT NULL DEFAULT 0,
  clients_served integer NOT NULL DEFAULT 0,
  pending_total integer NOT NULL DEFAULT 0,
  created_at timestamptz NOT NULL DEFAULT now(),
  UNIQUE(user_id, month, year)
);
```

### Grafico de Barras

- Usar apenas `View` e `Text` do React Native
- Calcular altura maxima como 150dp
- `height = (valor / maxValor) * 150`
- Se maxValor = 0, altura = 0

### Compartilhamento

- Usar `Share.share()` do React Native (nativo, sem biblioteca externa)
- Formato do texto:
  ```
  Relatorio GardenGreen - [Mes] [Ano]
  Receita: R$ X.XXX
  Despesas: R$ X.XXX
  Lucro Liquido: R$ X.XXX
  Servicos: X | Clientes: X
  ```

### Notificacao Push

- Usar Expo Push Notification API (token armazenado em `notification_preferences`)
- Mensagem: `"Seu relatorio de [mes] esta pronto! Voce ganhou R$ X.XXX"`
