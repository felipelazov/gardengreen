## Story

**As a** jardineiro autônomo,
**I want** receber uma notificação toda manhã com meus clientes do dia,
**so that** eu me prepare antes de sair de casa sem precisar abrir o app.

## Acceptance Criteria

```gherkin
AC-1: Notificacao enviada no horario correto
GIVEN usuarios com morning_enabled=true em notification_preferences
WHEN o cron executa as 05:30 BRT
THEN cada usuario recebe um push com os clientes agendados para hoje

AC-2: Mensagem com poucos clientes (ate 3)
GIVEN um usuario com 1-3 servicos hoje
WHEN o push e enviado
THEN a mensagem e "Bom dia, [Nome]! Hoje voce tem [X] clientes: [Nome1], [Nome2], [Nome3]"

AC-3: Mensagem com muitos clientes (mais de 3)
GIVEN um usuario com mais de 3 servicos hoje
WHEN o push e enviado
THEN a mensagem e "Bom dia, [Nome]! Hoje voce tem [X] clientes: [Nome1], [Nome2], [Nome3] e mais [Y]"

AC-4: Sem servicos hoje — sem push
GIVEN um usuario com nenhum servico agendado para hoje
WHEN o cron executa
THEN nenhuma notificacao e enviada para esse usuario

AC-5: Token invalido — ignorado silenciosamente
GIVEN um usuario sem expo_push_token valido
WHEN o cron executa
THEN o usuario e pulado sem erro critico (log de warning)

AC-6: Registro de push token no app
GIVEN um usuario autenticado abrindo o app pela primeira vez
WHEN as permissoes de notificacao sao concedidas
THEN o expo_push_token e salvo na tabela users
```

## Dev Notes

### Logica de Horario

- Cron `30 8 * * *` UTC = 05:30 BRT (UTC-3)
- Em horario de verao (BRST = UTC-2): ajuste necessario para `30 7 * * *`
- Recomendar configurar via variavel de ambiente ou verificar timezone do servidor

### Montagem da Mensagem

const names = services.map(s => s.clients.name);
const count = names.length;
let body: string;
